# New Developer Onboarding

Welcome to the Ryu Medha repository!\n\n## 1. What this project does\nRyu Medha is a study assistant and dashboard designed to help students track study time, academic attendance, and grades seamlessly. Users interact via WhatsApp for quick data entry and access a web dashboard for advanced analytics.\n\n## 2. Which applications are primary\n- The **WhatsApp Bot** (Supabase Edge Functions), which serves as the primary data entry point via natural language commands\n- The **Web Dashboard** (located in `web/`), which serves as the primary analytics and management interface\n- The **Supabase Backend** (located in `supabase/`), which powers both with database and serverless functions\n\n## 3. Which services are active\n- Supabase Edge Functions for the WhatsApp bot\n- The Next.js web application (`web/`)\n- Supabase database and authentication\n\n## 4. Which services are experimental/legacy\n- `web-clone/` and `web-legacy/` appear to be older or duplicate versions of the web frontend\n\n## 5. How to run locally\n- **Web**: `cd web && npm install && npm run dev`\n- **Supabase**: `supabase start` to run local Supabase instance\n- **Bot Functions**: `supabase functions serve` to test functions locally\n\n## 6. How to create a feature branch\nFollow the branching strategy: `git checkout -b feature/your-feature-name`.\n\n## 7. How to test changes\n- Front-end changes can be tested locally via the Next.js dev server\n- Bot changes can be tested using `supabase functions serve` and sending test messages\n- Database and RLS changes should be validated against the Supabase project\n\n## 8. How deployments work\n- **Frontend**: Deploy `web/` to Vercel or similar Next.js hosting (see [Web Deployment](../deployment/web.md))\n- **Supabase**: Deploy Edge Functions with `npx supabase functions deploy <function-name>` (see [Supabase Deployment](../deployment/supabase.md))\n- **Database**: Apply schema changes with `supabase db push`\n\n## 9. Common development workflows\n- Updating database schemas: Modify `supabase/ULTIMATE_CONSOLIDATED_SCHEMA.sql` and push to dev database\n- Adding a bot feature: Update `supabase/functions/whatsapp-bot/processor.ts` with new handler, add NLP intent to `nlp.ts`\n- Adding a dashboard feature: Add routes and components in `web/app/`\n\n## 10. Key documentation\n- **Bot Architecture**: [Edge Functions Bot Architecture](../deployment/edge-functions-bot.md)\n- **Bot Commands**: [Bot Features & Commands](../deployment/bot-features-commands.md)\n- **Data Flows**: [Bot Data Flows & Workflows](../deployment/bot-data-flows.md)\n- **Deployment**: [Deployment guides](../deployment/)
